# Задача: новый раздел «Ролики» в админке (ТЗ п.7)

## Контекст
В админке (Laravel) нужно создать новый раздел «Ролики» — управление короткими видео пользователей и их модерация. Дизайн-макет: вариант **10a** в Admin UI (полный экран: сайдбар + таблица роликов с фильтрами по статусу модерации).

Перед началом изучи существующий код админки (структуру контроллеров, вьюх/компонентов, роутов, сайдбара) и реализуй раздел **теми же паттернами**, что уже приняты в проекте (Blade/Livewire/Inertia/Vue — что уже используется), не меняя стек.

## Бэкенд / данные

Модель `Video` (или `Reel`) с полями:
- `video_path` — путь к видеофайлу (оригинал и сжатая версия);
- `title` — название;
- `tags` — теги (many-to-many `video_tag` или JSON-колонка — согласуй с тем, как теги уже сделаны в проекте, если сделаны);
- `user_id` — автор (FK на users);
- `status` — статус модерации: `pending` (на проверке, дефолт при загрузке) / `approved` / `rejected`;
- `reject_reason` — причина отклонения (nullable, заполняется при `rejected`);
- `likes_count` — счётчик лайков (денормализованный, поверх таблицы `video_likes`: `user_id` + `video_id`, unique);
- `views_count` — счётчик просмотров;
- `duration` — длительность в секундах;
- timestamps.

Правила:
- **Длительность ≤ 60 секунд** — валидация при загрузке (отклонять файл длиннее минуты).
- **Сжатие на бэкенде** — после загрузки видео ставится в очередь (job) на сжатие (ffmpeg); в админке до завершения можно показывать бейдж «обрабатывается», если это просто реализовать в текущем стеке.
- **Лимит по тарифу** — максимальное количество роликов пользователя определяется его тарифом (поле/связь тарифа уже есть в проекте — добавь в тариф параметр `video_limit`). Загрузка сверх лимита запрещена на уровне валидации.
- **Лайки** — эндпоинт для приложения: поставить/снять лайк (toggle), один лайк на пользователя на ролик.

## UI раздела (по макету 10a)

1. **Пункт в сайдбаре** — «Ролики» в группе «Главное» (после «Объявления»), иконка play-в-круге. Справа в пункте — счётчик роликов со статусом «на проверке» (маленький бейдж), чтобы модератор видел очередь не заходя в раздел.

2. **Шапка страницы** — надзаголовок «Главное», заголовок «Ролики» + пилюля с общим количеством («6 роликов»). Под заголовком — приглушённая строка-подсказка с иконкой часов: «До 1 минуты · видео сжимается автоматически при загрузке».

3. **Фильтры-чипы по статусу** (под шапкой, скругление 20px): «Все», «На проверке», «Одобрено», «Отклонено». В каждом чипе — счётчик с количеством; у счётчика «На проверке» — янтарный tint, «Одобрено» — зелёный, «Отклонено» — красный. Активный чип — залит акцентным цветом. Фильтр уходит в запрос (не только текущая страница, если есть пагинация).

4. **Таблица роликов**, колонки:
   - **Превью** — вертикальная миниатюра 38×56px (кадр из видео, 9:16), скругление 8px; поверх снизу — бейдж длительности («0:48») на полупрозрачном чёрном. Если кадр ещё не сгенерирован — заглушка с иконкой play.
   - **Ролик** — название (жирное, обрезается ellipsis) + строкой ниже теги через `#` акцентным цветом («#Недвижимость #Аренда»).
   - **Автор · Тариф** — имя автора + строкой ниже «Тариф Про · 12/30» (использовано/лимит по тарифу). Если лимит исчерпан — текст янтарным цветом и приписка «— лимит».
   - **Статус** — пилюля с точкой-индикатором: «На проверке» (янтарный), «Одобрено» (зелёный), «Отклонено» (красный). Для отклонённых — `title` с причиной («Причина: Признаки мошенничества»).
   - **Лайки** — иконка сердца + число.
   - **Просмотры** — иконка глаза + число.
   - **Дата** — дата загрузки.
   - **Действия** — иконки-кнопки 30×30px с `title`/`aria-label`:
     - для `pending`: «Одобрить» (галочка, зелёный tint-фон) и «Отклонить» (крестик, красный tint-фон);
     - для остальных: «Удалить» (корзина, при hover — красный).

5. **Модерация**:
   - «Одобрить» — статус `approved`, строка мгновенно обновляется.
   - «Отклонить» — выбор причины из существующего справочника причин отклонения (тот же, что для объявлений, или отдельный `video_reject_reasons` — согласуй с тем, как справочники уже устроены); причина сохраняется в `reject_reason`.
   - Действия доступны по правам модератора (та же система ролей, что в остальной админке).

6. **Пустое состояние** — «Нет роликов с этим статусом», по центру, приглушённым цветом.

7. **Hover строки** — лёгкая подсветка тем же токеном, что в остальных таблицах админки.

## Стиль
Тёмная тема, консистентная с остальной админкой: акцент индиго/фиолетовый (`#6D63F2`), зелёный `#4ADE80` (одобрено), красный `#F0554C` (отклонено/удаление), янтарный `#F5B14C` (на проверке / лимит), скругления 7–14px, шрифт Golos Text. Используй существующие цветовые токены темы; если янтарного токена «warn» ещё нет — добавь его в общую тему, не хардкодь в компоненте.

## Что важно не сломать
- Загрузку роликов делает мобильное приложение — админка только читает, модерирует и удаляет; API для приложения (загрузка, лента одобренных, лайки, просмотры) должен отдавать только `approved`-ролики обычным пользователям.
- Смена статуса и удаление — через подтверждённые действия с проверкой прав (policy/middleware, как принято в проекте).
- Удаление ролика должно чистить файлы (оригинал + сжатую версию + превью-кадр) и связанные лайки.
- Счётчики лайков/просмотров денормализованы — обновляй атомарно (increment), не пересчитывай на каждый запрос.
