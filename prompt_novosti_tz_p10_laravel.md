# Задача: привести раздел «Новости» в соответствие с ТЗ (п.10)

## Контекст
Форма создания/редактирования новости уже частично обновлена (тёмная тема, переключатель языка RU/TK, тумблер публикации, загрузка изображения с кропом). Этим тикетом нужно закрыть оставшиеся пункты ТЗ, которых раньше в форме не было.

## 10.1–10.2. Типы новостей + рекламная ссылка
- Поле «Тип» — два значения: **Обычная** / **Рекламная** (было три нерелевантных ТЗ значения — заменить).
- Если выбрана **Рекламная**, показать доп.блок «Ссылка рекламной новости»:
  - выбор сущности: **Профиль пользователя** / **Объявление** / **Товар**;
  - поле ID/ссылки на выбранную сущность.
- На бэкенде: добавить в таблицу новостей поля `type` (`regular`/`ad`), `ad_link_type` (`profile`/`listing`/`product`, nullable), `ad_link_id` (nullable, FK на соответствующую таблицу по типу — либо полиморфная связь `linkable_type`/`linkable_id`, если так принято в проекте).
- Валидация: `ad_link_type` и `ad_link_id` обязательны, если `type === 'ad'`; при `type === 'regular'` — не используются (обнулять при сохранении, если тип сменили с рекламной на обычную).

## 10.3. Редактор новостей с форматированием
- Поле «Содержание» — не простой textarea, а мини-редактор с базовым форматированием: **жирный**, *курсив*, **список**, **ссылка**, переносы строк.
- Хранить контент как HTML (санитизировать на бэкенде — разрешить только `<b>`, `<i>`, `<strong>`, `<em>`, `<ul>`, `<ol>`, `<li>`, `<a href>`, `<br>`, `<p>`; вырезать остальные теги/атрибуты, `on*`-обработчики, `javascript:`-ссылки).
- Использовать существующий в проекте WYSIWYG, если он уже подключён где-то ещё (например, для объявлений/описаний) — не тащить новую библиотеку, если задача решается тем, что уже есть в проекте.

## 10.4. Работа рекламных ссылок в push-уведомлениях
- При отправке push-уведомления о новости передавать вместе с payload: `type`, `ad_link_type`, `ad_link_id` (если рекламная).
- В мобильном клиенте (или как принято обрабатывать deep link в проекте) — по нажатию на push с рекламной новостью открывать нужный экран:
  - `profile` → экран профиля пользователя по `ad_link_id`;
  - `listing` → экран объявления по `ad_link_id`;
  - `product` → экран товара по `ad_link_id`.
- Если сущность не найдена (удалена/скрыта) — показать пользователю понятный экран-заглушку вместо ошибки/краша.

## Стиль
Тёмная тема, акцент `#6D63F2`, скругления 8–11px, шрифт Golos Text — консистентно с остальной админкой.

## Что важно не сломать
- Существующие обычные новости (`type = regular`) не должны требовать заполнения полей ссылки.
- Обязательность заголовка RU и остальная логика сохранения (черновик/публикация) не меняется.
