# Задача: добавить 3-й уровень «Районы» в раздел «Регионы и города» (Laravel)

## Контекст
В админке уже реализована иерархия **Регион → Город** (создание/скрытие/редактирование/удаление городов внутри региона). Нужно добавить ещё один уровень вложенности: **Регион → Город → Район**.
Пример: Марыйский (регион) → Йолотень (город) → Варашыл (район).

Перед началом изучи существующий код: миграции/модели/контроллеры/вьюхи для Region и City — новый уровень должен быть реализован **теми же паттернами** (именование, структура контроллеров, стиль валидации, фронтенд-стек — Blade/Livewire/Inertia/Vue, что уже используется), не изобретай новый подход.

## Модель данных
Добавить таблицу `districts` (или `rayons`, если в проекте уже принят транслит "район" в других сущностях — используй существующее соглашение):
- `id`
- `city_id` — FK на `cities`, `onDelete('cascade')`
- `name` — строка, обязательное
- `name_native` (если у City/Region есть аналогичное поле для туркменского названия — добавить и сюда для консистентности)
- `is_hidden` — boolean, default `true` (по умолчанию новая запись скрыта, как сейчас у городов/регионов)
- `sort_order` (если есть у City/Region — добавить и сюда)
- timestamps

Модель `District`:
- `belongsTo(City::class)`
- Модель `City`: добавить `hasMany(District::class)`

## Backend
Реализовать CRUD для районов по образцу существующего CRUD городов:
- Роуты вложенные под городом: `cities/{city}/districts` (store/update/destroy) + отдельный endpoint для toggle видимости (`is_hidden`).
- Валидация: `name` обязательно, уникально в рамках одного `city_id`.
- При удалении города — каскадно удалять его районы (см. FK выше).
- Если список городов/регионов отдаётся во фронтенд с counts (`cities_count`, и т.п.) — добавить аналогично `districts_count` у City.

## UI/UX (страница «Регионы и города»)
Расширить текущую 2-колоночную раскладку (Регионы | Города) до **3 колонок**: **Регионы → Города → Районы**.

Поведение:
- Клик по региону в 1-й колонке — во 2-й колонке подгружается список его городов (без перезагрузки страницы).
- Клик по городу во 2-й колонке — в 3-й колонке подгружается список его районов.
- Пока регион/город не выбран — соответствующая следующая колонка показывает плейсхолдер: «Выберите регион» / «Выберите город»; если выбран, но список пуст — «Нет городов» / «Нет районов».
- Кнопка «+ Добавить» есть в шапке каждой из 3 колонок. Для колонок «Города» и «Районы» она неактивна (серая), пока не выбран родитель (регион/город соответственно).
- Клик «Добавить» открывает **строку ввода прямо в списке** (без модалки): текстовое поле + кнопки «Сохранить» / «Отмена». Это должно работать одинаково на всех 3 уровнях.
- У каждой строки (регион/город/район): название + (для региона/города) подстрока вида «Mary · 3 города» / «Ýolöten · 2 района»; и три действия справа — тоггл видимости («Скрыт»/«Показан»), редактировать (карандаш, инлайн — заменяет название на текстовое поле с «Сохранить»/«Отмена»), удалить (корзина, красный акцент).
- Заголовок 2-й и 3-й колонки — с хлебной крошкой: «Города — Марыйский», «Районы — Йолотень».
- Выбранная строка в колонке подсвечивается мягким акцентным фоном (tint), без изменения структуры.

## Стиль
Тёмная тема, консистентная с остальной админкой: тёмный фон, карточки чуть светлее фона, акцент — индиго/фиолетовый (`#6D63F2` / `#4F46E5` в зависимости от темы), скругления 8–14px, шрифт Golos Text. Используй уже существующие в проекте цветовые токены/переменные темы вместо новых значений.

## Что важно не сломать
- Существующий функционал регион→город должен продолжать работать как раньше.
- Скрытие/удаление региона или города должно каскадно скрывать/удалять вложенные районы (или хотя бы явно это обрабатывать — уточни у меня, если в проекте уже есть своя логика на этот счёт).
